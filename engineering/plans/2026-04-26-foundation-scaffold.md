# Arc Foundation Scaffold Implementation Plan

> **For implementers:** REQUIRED SUB-SKILL: Use superpowers:test-driven-development where code behavior is introduced. Every implementation task must commit its initial work before review, then commit any review fixes separately. Report all commit SHAs.

**Goal:** Establish the first production-oriented Arc/OpsCanvas monorepo foundation without overbuilding product features.

**Architecture:** This repository starts as the Arc engineering home, but the first build substrate is the shared OpsCanvas foundation described in the local product docs. We will create a conservative monorepo shape with Python services/packages, a future Next.js web app location, and infrastructure schema placeholders. The canonical runtime model lives in a shared Python package first, then API, plugin, storage, and web layers depend on it.

**Tech Stack:** Python 3.12, `uv` workspace, FastAPI, Pydantic v2, pytest, Ruff, mypy, pnpm workspace, Next.js/React/TypeScript later, ClickHouse/Postgres/Redis schema files.

## Task 2: Canonical Run/Span/Event Contracts

**Purpose:** Define the first versioned internal data contracts that every runtime plugin and ingest path will emit.

**Files:**
- Modify: `packages/opscanvas-core/pyproject.toml`
- Create: `packages/opscanvas-core/src/opscanvas_core/schema_versions.py`

- Create: `packages/opscanvas-core/tests/test_schema_versions.py`
- Create: `packages/opscanvas-core/tests/test_redaction.py`

**Contract Requirements:**
- Use the SDK reuse policy pattern: `CURRENT_SCHEMA_VERSION`, `SCHEMA_VERSION_SUMMARIES`, `SUPPORTED_SCHEMA_VERSIONS`.
- Define `RunStatus`: `succeeded`, `failed`, `interrupted`, `suboptimal`, `running`.
- Define `SpanKind`: `agent`, `model_call`, `tool_call`, `handoff`, `guardrail`, `mcp_list`, `sandbox_op`, `retry`, `custom`.
- Define Pydantic models for `Usage`, `Run`, `Span`, and `SpanEvent`.
- Models must serialize deterministically with aliases suitable for JSON ingestion.
- Add simple ID generation helpers that produce prefixed IDs for run/span/event.
- Add a minimal regex redactor for email and US phone numbers. It should be explicit that this is not the final compliance-grade redactor.

**Verification Commands:**
- `uv run pytest packages/opscanvas-core/tests -q`
- `uv run ruff check packages/opscanvas-core`
- `uv run mypy packages/opscanvas-core/src`

**Expected Result:** The core package owns the canonical v0 run/span/event shape without importing runtime-specific SDKs.
