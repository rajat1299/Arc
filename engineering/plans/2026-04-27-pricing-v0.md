## Pricing Semantics

- Catalog prices are USD per 1 million tokens.
- `compute_cost(usage, model, provider)` returns a cost breakdown when both provider/model are known and usage contains billable tokens; otherwise returns `None`.
- `input_tokens` is treated as total input tokens, including `cached_input_tokens` when that field is present. Bill uncached input as `max(input_tokens - cached_input_tokens, 0)` at the input rate and cached input at the cached-input rate when the catalog has one. If the catalog has no cached-input rate, bill all input at the normal input rate.
- `output_tokens` is billed at output rate. `reasoning_tokens` is not billed separately in v0 unless a future catalog entry explicitly adds a separate rate; Google's "thinking tokens included in output price" stays covered by output tokens.
- Unknown providers/models, missing usage, and usage with no billable token counts return `None`.
- Use `Decimal` internally and round final USD totals to 10 decimal places for stable API/test output.

## Task 2: API Cost Fallback Integration

**Purpose:** Let API summaries and metrics use computed cost only when reported cost is absent.

**Files:**
- Modify: `services/api/src/opscanvas_api/routes/runs.py`
- Modify: `services/api/tests/test_runs.py`

**Requirements:**
- Keep `usage.cost_usd` authoritative. If a run has reported `usage.cost_usd`, summaries and metrics must use that value.
- If a run has no reported `usage.cost_usd`, compute cost from model spans where possible:
  - provider from span attributes `provider`
  - model from span attributes `model` or `agent.model`
  - usage from the span
- If no span cost can be computed, optionally attempt run-level metadata `provider` and `model` with run usage.
- Unknown/unpriced runs contribute `0` to aggregate cost, as today missing costs do.
- Preserve existing response model names unless adding fields is necessary. Prefer keeping `RunSummary.cost_usd` and `RunMetrics.total_cost_usd` as effective reported-or-computed costs for v0.
- Do not mutate `Run` or `Usage`; this is a read-time fallback.
- Tests must cover:
  - reported run cost still wins over computed span costs.
  - missing run cost can be computed from OpenAI span usage.
  - unknown model with missing run cost leaves summary cost `None` and metrics cost unchanged.
  - mixed reported/computed/unpriced aggregate totals.

**Verification Commands:**
